一、项目概述
1. 目标硬件：ESP8266-01S 模块（1MB Flash、单核、外设精简、引脚稀少、内存资源受限）
2. 开发框架：ESP8266 RTOS SDK，原生搭载 FreeRTOS 实时操作系统，原生支持多任务并发调度，纯 C 语言编码，满足业务分层、独立任务运行需求
3. 工程配置：通过 platformio.ini 统一管理编译、烧录、固件参数；受硬件1MB Flash限制，取消双分区OTA、取消NVS分区，仅支持单分区固件烧录与运行
4. 构建工具：统一使用根目录 build.py 脚本完成编译、上传、串口监控、代码清理，不使用自定义编译脚本
5. 版本管理：Git 版本管控，项目开发版本从 v0.0.1 开始迭代，迭代记录可追溯
6. 硬件裁剪适配：硬件无原生I2C/SPI外设、可用引脚极少，项目仅保留开门器核心业务功能，彻底裁剪所有冗余外设、冗余服务、冗余驱动，保证固件体积最小、运行最稳定。

二、分层架构设计（标准五层架构 + 公共层）
项目严格沿用工业级五层分层架构，单向依赖、上层调用下层、禁止跨层、禁止反向调用，基于ESP8266-01S硬件特性精简无用层级与模块，架构标准化、低耦合、易维护。
┌─────────────────────────────────────────────────────┐
│  App 应用层                                         │  开门业务逻辑、状态机、权限判断、场景联动
├─────────────────────────────────────────────────────┤
│  Service 系统服务层                                 │  WiFi、MQTT、Web配网、参数存储
├─────────────────────────────────────────────────────┤
│  Driver 外设驱动层                                  │  LED状态、舵机开门、红外发射、按键检测
├─────────────────────────────────────────────────────┤
│  BSP 板级支持层                                     │  GPIO、UART 底层硬件统一封装
├─────────────────────────────────────────────────────┤
│  Common 公共基础层                                  │  全局配置、日志、错误码、工具函数
├─────────────────────────────────────────────────────┤
│  ESP8266 RTOS SDK 框架层                            │  系统底层（框架自带，禁止修改）
└─────────────────────────────────────────────────────┘

架构补充说明：RTOS 任务不属于单一业务层级，为横向贯穿所有层级的调度单元，统一独立存放于 app_task 目录、统一调度管理，实现业务并发解耦。
【各层级详细职责 & ESP8266 专属适配规则】
1. Common 公共层（全局最底层，所有层级可调用）
- 统一存放全局硬件参数、业务参数、任务参数、功能开关
- 提供统一错误码、分级日志宏、通用数据类型定义
- 封装轻量工具函数：字符串处理、数据转换、简单校验
- 无任何业务逻辑、无硬件动作，纯基础支撑模块
- 适配优化：大幅精简冗余配置、冗余宏定义，适配ESP8266小内存、小Flash资源限制，避免内存浪费、固件臃肿
2. BSP 板级支持层
- 仅封装ESP8266必备外设：GPIO、UART，彻底裁剪I2C、SPI、PWM等硬件不支持的外设驱动
- 屏蔽ESP8266 SDK底层差异，向上层Driver、Service提供统一、标准化硬件接口
- 硬件适配仅修改此层，上层业务代码完全无需改动
- 严格禁止包含业务逻辑、状态判断、功能决策
3. Driver 外设驱动层
- 仅适配ESP8266-01S实际可用外设：板载LED、舵机开门模块、红外发射、按键检测
- 所有硬件操作通过调用BSP层接口实现，禁止直接操作寄存器、禁止直接调用底层SDK
- 单个器件驱动独立封装、可插拔、互不依赖，方便功能裁剪与迭代
4. Service 系统服务层（已完成ESP8266精准裁剪）
- 保留核心刚需服务：WiFi连接管理、MQTT远程通信、Web配网服务、本地参数存储
- 彻底裁剪ESP32专属、ESP8266不兼容、项目无用功能：NTP网络时间、OTA远程升级、蓝牙服务、高级NVS存储、多级系统管理服务
- 所有服务自带状态机管理，仅提供初始化、状态查询、数据收发接口，不参与业务决策
- 适配1MB Flash精简设计，去除冗余缓存、冗余校验、冗余后台任务
5. App 应用层（最上层业务核心）
- 实现开门器完整产品业务逻辑：开门状态机、指令权限校验、MQTT指令解析、场景联动、状态上报
- 统一调度Driver外设驱动、Service系统服务，组合实现完整产品功能
- 禁止直接操作硬件、禁止跨层调用底层接口、禁止直接调用SDK原生API

---
三、核心设计约束（代码强制遵守）
1. 所有用户业务参数、硬件引脚、阈值参数，统一集中在全局配置区（common/config.h），禁止代码硬编码
2. 严格单向分层调用，杜绝跨层、反向依赖，保证代码低耦合、高可维护
3. 所有并发业务基于FreeRTOS独立任务实现，任务间仅通过队列、事件组通信，禁止直接函数互调
4. 适配ESP8266-01S硬件限制，不新增冗余功能、不引入大型库、不开启无用服务
5. main.c 的 user_init 仅做初始化与任务创建，不写任何业务逻辑
6. 所有代码使用纯 C 语言编写，遵循 C99 标准，函数使用 ICACHE_FLASH_ATTR 宏标注放置于 Flash
